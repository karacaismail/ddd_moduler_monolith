Tahta — Eksik Tamamlama

Sütunlarda eksik olanlar

15 üründen (Odoo/SAP/Frappe/WP/Drupal/Shopify dahil) gerçekten ihtiyaç duyulacak primitive'lerin senin mevcut tahtana eklenmesi gereken karşılığı. Sade liste, tek satır gerekçe.

Kritik — eksik kalırsa framework olmaz Normal — değer katar, sonradan eklenebilir

Kum Tanesi +18 öneri

Atomik primitive katmanı. Her şeyin oluşturucu blokları. Senin Field, Log, Handler, Security dışında DocType engine'i besleyecek atomik tipler çoğunlukla eksik.

  • KritikType SystemField'ın altında: scalar/composite tip tanımı, validator, coercion. Her field bir type'a referans verir.
  • KritikID GeneratorUUID v7, ULID, snowflake, tenant-prefix'li ID. Distributed sistemde sequence yetmez.
  • KritikMoney / DecimalFloat yasak (yuvarlama hataları → muhasebede para kaybı). Decimal + currency code çifti.
  • KritikDateTime / TimezoneUTC store + IANA timezone display. HRMS effective-dating bunsuz olmaz.
  • KritikReference / LinkDocType'a foreign key. Cascade kuralı, orphan kontrolü, tenant-scope check.
  • KritikCounter (hot)Like/view/follower. Backend swap edilebilir (DB → Redis → sharded).
  • KritikVersion / ETagOptimistic concurrency — aynı record'u iki kullanıcı eşzamanlı değiştirirse çakışma.
  • NormalHash / ChecksumDosya integrity, idempotency key, content-addressable storage (Drive).
  • NormalSlugURL-safe identifier, Türkçe karakter normalizasyonu, uniqueness scope.
  • NormalPhone (E.164)Country code parsing, format, validation. Twilio/NetGSM gönderiminde şart.
  • NormalEmailValidation + DNS MX check + disposable detection.
  • NormalURLValidation + canonical form + scheme allowlist.
  • NormalGeo Coordinatelat/lng + PostGIS bridge. Real estate (#1), Field Service.
  • NormalAddress (atomic)Country/region/city/postal yapısı + locale formatting.
  • NormalColorTheme, calendar event, tag — UI primitive.
  • NormalTagPolymorphic label, çok-tag yapısı (Taxonomy'den farklı: hiyerarşisiz).
  • NormalSoft DeleteTrash, restore, hard-delete policy. KVKK silme talepleri için altyapı.
  • NormalEncryption FieldPII/secret field-level encryption, key rotation. Security primitive'in detayı.

Küçük Taş +12 öneri

Foundation katmanı. Burada en kritik eksik var: DocType / Schema Engine. Sen Kernel'in altında Worker/Cron/Agent yazdın ama asıl framework'ü framework yapan motor tahtanın dışında kalmış.

En kritik eksik

DocType engine, Event Bus + Outbox ve Polyglot Runtime Bridge yok. Bunlar olmadan Frappe/Drupal/Odoo benzeri "metadata-driven, plugin-able framework" hedefini taşıyamazsın. Layer 0 kernel.

  • KritikDocType / Schema EngineTahtanın merkezi olmalı. Field'lardan DocType, DocType'tan tablo + REST + form + permission otomatik. Frappe DocType, Salesforce Object, Strapi Content-Type.
  • KritikEvent Bus / Pub-SubModule'lerin loose-coupled konuşma omurgası. In-process → Redis Streams → Kafka aynı interface.
  • KritikOutboxDual-write problem'in tek doğru çözümü. Save pipeline'a örülü.
  • KritikPolyglot Runtime BridgeHook contract Python + TS aynı interface. Embedded (PythonMonkey/STPyV8) / Sidecar (Node gRPC) / WASM (Extism) — üç runtime.
  • KritikPlugin / Extension RegistryHook noktaları, manifest, dependency resolution. WordPress hooks/filters, Odoo `__manifest__.py`, Backstage extension points.
  • KritikMigration EngineDocType değişimi → schema migration → veri taşıma. Versiyonlu, geri alınabilir, dry-run.
  • NormalCache (interface)In-process LRU → Redis → CDN (HTTP cache). Driver pattern.
  • NormalRate LimitingPer-user, per-IP, per-tenant, per-endpoint. Brute force koruması.
  • NormalFeature FlagTenant-bazlı, rollout, A/B. Plan-bazlı feature gating (SaaS).
  • NormalSettings / ConfigTenant config, framework config, module config hiyerarşisi. Hot reload.
  • NormalSecrets VaultAPI key, OAuth secret saklama. Env'den ayrı, rotate edilebilir.
  • NormalHealth / ReadinessK8s probe, dependency health (DB, Redis, Kafka up?). Operasyon zorunluluğu.
Auth'un altında bir not

Authorization'ın altına RBAC da ekle. ReBAC modern ama klasik admin/editor/viewer rolleri RBAC. Üçü birlikte (RBAC + ReBAC + ABAC) gerçek hayatta.

Önerilen sütun yeniden adlandırma

Tahtanın yapısı sağlam ama iki düzeltme:

  • Normal"Identity" Küçük Taş'ta zaten Auth altında var, Orta Taş'tan kaldır.Duplicate. Auth.Identity tek noktada kalsın.
  • Normal"File Storage" Küçük Taş'ta — yer doğru, ama içine Versioning, Lifecycle, Signed URL ekle.Drive (#7) ürünü için CAS/dedup ayrı module, ama temel versioning core.
  • Normal"ECA Engine" Küçük Taş'ta, "ECA Rules" Orta Taş'ta — bu doğru ayrım, koru.Engine = motor (kernel-yakını), Rules = kullanıcı-tanımlı kural seti (module).
  • Normal"Block" + "Section" Orta Taş'ta — CMS authoring'in altına nest et.Authoring → Block → Section hiyerarşi açık olsun.
